# 로컬 상인을 위한 전단지P 메리트/디메리트 분석

## 핵심 발견사항

### 포인트 경제 구조 (코드 검증됨)
- 사업자는 포인트 예산(point_budget)을 충전하여 유저에게 보상 제공
- 공유/퀴즈/QR 방문 시 점진적 차감
- **비즈니스 모델 한계**: 사업자가 실제 돈으로 충전 → 포인트로 변환하는 결제 통합 **미구현**
  - charge-points API는 수동으로 point_budget만 증가
  - 실제 결제 게이트웨이(결제모듈) 없음 → 현재는 데모/테스트 상태

### 사업자 기능 구현 상태
- 대시보드: 통계/예산/퀴즈/QR 관리 완성도 높음
- **미흡한 부분**: ROI 추적 부재, 실시간 분석, 광고 타게팅 옵션 없음

### 타겟 사업자 유형
- 체인점/대형마트보다 **개별 소상공인** (카페/식당/뷰티)에 더 적합
- 이유: 종이 전단지 대비 비용 절감, 실시간 고객 추적 가능
- 현재 시드 데이터는 대형업체(이마트, 스타벅스, 올리브영) → 부정합 문제

## 메리트 (강점)

### 1. 비용 절감
| 항목 | 종이 전단지 | 전단지P |
|------|-----------|---------|
| 인쇄비 | 10만원~50만원/회 | 0원 |
| 배포비 | 5만원~10만원 | 0원 |
| 디자인 | 외부 용역 비용 | 셀프 (온라인) |
| **월 예상 비용** | **50~100만원** | **포인트 충전만** |

### 2. 타겟 고객 도달의 정확성
- 네이버/카카오맵과 달리 **능동적 고객 참여** 기반
- 긁기(scratch) → 퀴즈 → 방문 인증 = **3단계 engagement 펀넬**
- 비교: 종이 전단지 (도달율 불명확) vs 배달앱 (시스템 장악 불가)

### 3. 고객 재방문 유도 메커니즘
- **QR 방문인증 1일1회 제한** → 연속 방문 유도
- 퀴즈 참여율 추적 가능 (현재 43% 수준 시드)
- 포인트 → 기프티콘 → 재구매 사이클

### 4. 마케팅 비용 대비 성능 측정
```
공유 수 (클릭율) → 퀴즈 참여율(%) → 방문 인증(재방문율)
25,678 views → 60% quiz → 300 visits = ROI 측정 가능
```
**기존 광고**: "효과 불명확" / **전단지P**: "모든 액션 추적"

### 5. 오프라인 연동 (QR)
- 실제 매장 방문 검증 (GPS 아님 → 조작 방지)
- 당근마켓 같은 오프라인 우월성 활용

## 디메리트 (약점)

### 1. 핵심 비즈니스 로직 미흡

#### 문제 1: 결제 시스템 미구현
```javascript
// server/routes/business.js 112-142 참고
router.post('/charge-points', authMiddleware, async (req, res) => {
  // 포인트만 더하고, 실제 결제(신용카드/계좌이체) 없음
  await txDb.prepare('UPDATE users SET point_budget = point_budget + ?').run(amount)
  // PG(Payment Gateway) 연동 전혀 없음
})
```
**현실의 문제**:
- 사업자가 실제 돈을 내지 않음 → 시스템 신뢰도 하락
- 충전 과정이 불명확 → "정말 청구되나?" 의심
- 현재는 관리자가 수동으로 충전해주는 상태

#### 문제 2: 출금 기능 제외
```
사용자는 포인트 출금 가능 (최소 1,000P)
사업자는 포인트 출금 불가능
```
→ 사업자 잔여 예산은 **환불/회수 불가** = 일방적 손실 위험

#### 문제 3: 사업자 등급/승인 프로세스 허술
```sql
ALTER TABLE users ADD COLUMN business_approved INTEGER NOT NULL DEFAULT 0
```
- 누구나 "business" 역할로 가입 가능
- 승인 프로세스 미구현 → 무분별한 사업자 증가 → 사용자 경험 악화

### 2. 포인트 경제 불균형

#### 문제: 사업자 예산 소진 위험
```
공유 포인트: 30~100P
퀴즈 포인트: 10~50P
QR 방문: 100~500P (가장 비쌈)

1개 전단지 예산 10,000P →
  · 공유만: 100~333명 도달
  · QR까지: 최소 20명 방문 후 소진
```
→ **작은 예산으로 큰 기대는 어려움**

#### 현실: 1천원 벌기 위해 몇 포인트?
- 배달앱 광고비: 일반음식점 월 10~20만원 (고정)
- 전단지P: 월 얼마? → **정해진 가격이 없음** (충전 선택)
  → 사업자 입장: "얼마 써야 효과 볼까?" 불명확

### 3. 앱 사용자 수(MAU) 의존성 극도로 높음

#### 문제: 닭과 달걀 싸움
```
사업자 관점: "사용자 많아야 가치"
사용자 관점: "전단지 많아야 쓸 가치"
```

**현재 시드 데이터**:
```
전단지: 20개 (대형 브랜드 위주)
예상 사용자: 소수 (데모 수준)
```

→ "사업자들이 참여해도 도달할 사용자가 극도로 적음"
→ 사업자 입장: "이 앱 사용자가 몇 명이나 되나?" 신뢰 부족

### 4. 기존 경쟁 서비스 압도

| 서비스 | 사용자 기반 | 사업자 비용 | 기능 |
|--------|-----------|----------|------|
| **배달앱** (배달의민족) | 수백만 | 월 10~50만 | 주문/배송 연동 |
| **플레이스** (네이버) | 수백만 | CPC/CPA 모델 | 리뷰·사진·예약 |
| **당근마켓** (소상공인) | 100만+ | 월 0~10만 | 지역 타게팅 |
| **전단지P** | 수천~수만? | 포인트 충전? | 전단지·포인트 |

→ 사업자 선택: **기존 플랫폼이 더 명확하고 검증됨**

### 5. 사업자 대시보드 기능 부족

#### 실제 구현된 기능
- ✅ 전단지 CRUD
- ✅ 퀴즈 등록 (3~5개, 포인트 10~50)
- ✅ QR 생성 및 통계
- ✅ 포인트 예산 확인
- ❌ A/B 테스팅 도구
- ❌ 시간대별 분석 (예: 점심 시간 방문율)
- ❌ 고객 세분화 (예: 재방문율, 구매전환율)
- ❌ 경쟁 상인 벤치마킹
- ❌ 자동 포인트 최적화 (예: 낮은 참여율 → 자동 증액)

### 6. 신뢰도 문제

#### 문제 1: "정말 유저가 받을까?"
- 사업자: "포인트 예산 충전했는데, 실제 사용자가 기프티콘으로 환전할까?"
- 현재 기프티콘 교환 구현은 있지만, **실제 카카오톡 발송** 메커니즘 불명확
- → 사업자가 "포인트 낭비"로 느낄 가능성 높음

#### 문제 2: "앱이 정말 지속될까?"
- 사업자: "스타트업이면 언제 서비스 종료될 텐데?"
- 장기적 신뢰 구축 필요

## 로컬 상인 유형별 적합도 평가

### 적합도 높음 (별 5개)
- **카페** (스타벅스, 이디야): 주기적 방문 유도 용이
- **음식점** (치킨, 버거): QR 방문인증 활용 최고
- **편의점**: 낮은 진입장벽, 빠른 ROI 기대

### 적합도 중간 (별 3개)
- **뷰티** (올리브영): 고객 단가 높지만, 계절성 약함
- **의류** (ABC마트): 할인 선호도 높음
- **마트**: 이미 고객이 많아서 신규 고객 유도 효과 낮음

### 적합도 낮음 (별 1개)
- **병원/약국**: 큰 할인 불가능, 고객 선택의 여지 적음
- **학원**: 계약 기반이라 포인트 효과 미미
- **부동산**: 리드 구조 전혀 다름

## 상인 유치 전략 권장사항

### 1순위: 성과 기반 모델 도입
```
현재: "포인트 충전 후 기원"
개선: "CPA(Cost Per Action) 모델"
  예) 실제 방문 1건당 1,000원 수수료
  → 사업자 심리: 결과 기반이라 안심
```

### 2순위: 무료 체험
```
신규 사업자: 초기 10,000P 무료 크레딧
  · 성과 측정 후 충전 의사 결정
  · 초기 진입장벽 낮춤
```

### 3순위: 로컬 영업팀 구성
```
당근마켓 성공 사례: 지역 영업 담당자가 소상공인 직접 방문
전단지P 현황: 온라인/앱 기반만 (오프라인 콜드 체계 없음)
```

### 4순위: 포인트 환율 투명화
```
현재: "10,000P 충전 → 몇 명 도달?" 불명확
개선: "평균 도달: 100명, 재방문율: 30%, 예상 고객: 30명"
```

## 결론

**현재 상태**: 로컬 상인 유치하기 어려움
- 비즈니스 로직 미흡 (결제 없음, 출금 없음, 승인 없음)
- 사용자 수 부족 → 도달 효과 의문
- 기존 플랫폼 (배달앱, 플레이스, 당근마켓)과의 경쟁에서 열위

**해결책**:
1. 결제 통합 (최우선)
2. 실제 사용자 기반 확보 (MAU 증가)
3. 오프라인 영업 시작
4. 투명한 성과 지표 공개
5. CPA 모델 테스트

**타겟 사업자 (현재 최적)**:
- 카페/음식점 사장 중 **기술에 친화적인 분**
- 월 5~10만원 마케팅 예산 있는 **소상공인**
- **새로운 고객 유입**이 급한 신규 매장
