# 전단지P 포괄적 기획 리뷰 (v4.3)

## 📊 서비스 핵심 분석

### 현재 포인트 경제
- **3가지 적립 채널**: 전단지 긁기(share_point), 퀴즈(10~50P), QR 방문(100~500P)
- **포인트 사용**: 기프티콘 교환(3,000~15,000P), 현금 출금(1,000P 이상, 7일 경과)
- **주목점**: 긁기와 퀴즈는 낮은 포인트, QR 방문이 주요 수익원. 일일 최대 적립 제한 없음

### 사용자 역할 분화
- **user(일반)**: 포인트 적립/소비 전체 가능
- **business(사영업자)**: 전단지 관리, 포인트 예산 제어, 포인트 획득 불가
- **admin(슈퍼어드민)**: 별도 admin.html 대시보드

### 주요 보안 메커니즘
- Device fingerprinting (다중계정 방지, 최대 2계정/기기)
- Scratch session token (긁기 유효성 검증)
- Bot detection (최소 3초 긁기 시간 강제)
- Share history UNIQUE 제약 (중복 공유 방지)
- 출금 지연 (가입 7일 후)

## 🔍 현재 상태 분석

### 아키텍처 강점
- Hash 기반 SPA로 가볍고 빠른 네비게이션
- Lazy loading + Suspense로 초기 로딩 최적화
- SQLite/Turso 듀얼 구조로 로컬/프로덕션 환경 분리
- JWT + OAuth(카카오/구글) 안정적 인증

### 현재 구현된 기능들
1. **MainPage**: 카테고리 필터, 검색, 무한스크롤, 북마크
2. **DetailPage**: 전단지 상세보기, 상품 목록, 자동 퀴즈 모달
3. **ScratchCard**: Canvas 기반 80% 긁기 복권 (게스트는 맛보기 1회 제한)
4. **QuizModal**: 객관식/주관식(입력형) 퀴즈, 즉시 결과
5. **QrScanPage**: BarcodeDetector API 기반 스캔, 1일 1회 제한
6. **GiftShopPage**: 12개 기프티콘 카테고리별 정렬, 전화번호 입력 후 교환
7. **MyPage**: 포인트/랭크 표시, 3가지 포인트 내역 탭, 출금/문의/북마크 관리
8. **AdminPage**: 유저/전단지/포인트/자영업자 관리 (별도 인터페이스)
9. **PWA 지원**: 설치 배너, 푸시 알림, 오프라인 지원 시작

## 💡 주요 발견 사항 (문제점 + 기회)

### 🔴 Critical 문제들

#### 1. 게스트 사용자 경험의 이중 모달 문제
**현상**: 게스트가 1회 긁기 후 다시 긁으려 하면
- 첫 번째: "로그인이 필요해요" 모달 (handleGuestBlock)
- 이를 닫으면: 다시 같은 전단지를 클릭해야 함
- UX: 혼란스러운 상태 전환

**영향**: 게스트 → 로그인 전환율 저하

**개선안**: 게스트 블록 모달을 "맛보기 완료" 모달로 통합하고, 한 번에 로그인으로 이동

---

#### 2. 퀴즈 표시 로직의 일관성 문제
**현상**: DetailPage에서 quizAttempted 배지가 표시되지만,
- 퀴즈에 실패(오답)해도 시도 기록이 남음
- 재방문 시 자동으로 "참여 완료" 상태
- 사용자는 "다시 풀어볼 수 없는가?" 혼동

**영향**: 포인트 기대치 관리 미흡

**개선안**:
- 퀴즈 히스토리에 "정답/오답" 명시
- "정답했으면 다시 풀 수 없음" 명확한 메시지
- 오답 시에도 재시도 가능하도록 변경 고려 (포인트 재설정)

---

#### 3. 출금 시스템의 UX 마찰
**현상** (MyPage):
- 출금 신청 후 "대기중" 상태 표시
- 관리자 승인 대기 시간 명시 없음 ("언제 받나요?")
- 출금 거절 시 사유 설명 부족
- 입금 실패(계좌번호 오류) 처리 불명확

**영향**: 사용자 신뢰 하락, 1:1 문의 증가

**개선안**:
- 출금 예상 시간 안내 (ex: "3~5 영업일")
- 거절 사유 상세 표시
- 재신청 절차 명확화

---

#### 4. 비로그인 사용자의 포인트 심화 이해 부족
**현상**:
- 게스트가 ScratchCard에서 긁기를 끝내고 confetti 볼 때
- "로그인하면 포인트가 실제로 적립돼요"라는 메시지
- 하지만 다른 전단지는 긁을 수 없음 (1회 제한)
- 포인트 심화 구조(기프티콘/출금)를 모름

**영향**: 로그인 의욕 부족

**개선안**:
- 게스트 confetti 모달에서 "포인트로 뭘 할 수 있나?" 시각적 표시
- 기프티콘 교환/출금 아이콘 포함

### 🟡 Important 개선점들

#### 1. 포인트 경제 밸런싱
**현상**:
- 공유: 보통 10~100P (사업자 설정)
- 퀴즈: 10~50P
- QR 방문: 100~500P
- 기프티콘: 3,000~15,000P 필요 (최소 30~150회 긁기?)

**문제**:
- QR 방문이 주요 수익원이지만, QR이 있는 전단지는 제한적
- 퀴즈 객답률이 명확하지 않음 (서버 검증 로직 확인 필요)
- 출금까지 가려면 심한 그라인딩

**개선안**:
- 일일 기프티콘 권장 목표량 시각화 ("이 속도면 X일 안에 교환!")
- 퀴즈 힌트 시스템 추가 (3개 보기 → 1개 제거 기회)
- 스트릭 보너스 (7일 연속 로그인 +50P 등)

---

#### 2. 오류 메시지의 친화성
**예시**:
- "이미 공유한 전단지입니다." (DetailPage에서 공유 재시도)
- "이 전단지의 포인트 예산이 소진되었습니다." (사용자 입장에선 왜?)
- "유효하지 않은 긁기 세션입니다." (사용자는 뭘 잘못했나?)

**개선안**:
- 사용자 친화적 메시지로 변경
- 원인 + 해결책 함께 표시

---

#### 3. 배너/탭 색상 가독성
**현상**:
- 카테고리 탭의 텍스트 색상이 배경과 대비 부족 (특히 다크모드)
- 카테고리 배지 배경색이 흐릿함

**개선안**:
- WCAG AA 기준 명도 대비 확인 (4.5:1 이상)
- 다크모드에서 별도 색상 팔레트

---

#### 4. 검색 기능의 실용성
**현상** (MainPage):
- 검색어 입력 중 결과 수만 표시 ("{query} 검색 결과 {totalCount}개")
- 검색 오버레이가 flyer-list와 겹쳐서 혼란
- 검색 결과가 없어도 오버레이 닫고 다시 열어야 함

**개선안**:
- 입력 중에 autocomplete 제안 표시
- 검색 결과를 오버레이 내 별도 섹션에 표시
- "검색어 없음" 상태에서 최근 검색 또는 인기 태그 표시

---

#### 5. QR 스캔 실패 처리
**현상** (QrScanPage):
- BarcodeDetector가 지원 안 될 때 폴백 없음
- 카메라 권한 거부 시 메시지 부족
- 저조도 환경에서 스캔 실패 가능성

**개선안**:
- 카메라 권한 명확한 안내 모달
- "QR 코드 수동 입력" 텍스트 입력란 추가
- 스캔 실패 시 UI 피드백 개선

---

#### 6. 기프티콘 교환 후 추적 부족
**현상** (GiftShopPage):
- 교환 후 "발송 완료" 상태만 표시
- 발송 상태가 카카오톡인지 문자인지 불명확
- 기프티콘 수령 여부 확인 불가

**개선안**:
- "발송됨" → "카카오톡 발송" / "문자 발송" 구분
- 발송 재시도 옵션
- 기프티콘 유효기간 알림

## 🚀 신규 아이디어 (전략적)

### 1. "이벤트 전단지" 시스템
**목표**: DAU 증대, 일일 로그인 습관 형성

**개요**:
- 매일 00:00에 "오늘의 전단지" 5개 공개
- 이들은 모두 QR 방문 가능 (100P/일)
- 5개 모두 스캔하면 보너스 200P

**예상 효과**:
- 일일 방문 유도
- 오프라인 매장 트래픽 증가
- 자영업자의 유료 "이벤트 전단지" 출시 가능 (월 10,000P 예산)

**구현 복잡도**: 낮음 (스케줄링 + 필터링)

---

### 2. "포인트 랭크" 시스템 + 랭크 리워드
**목표**: 경쟁심리를 통한 참여도 증대

**개요**:
- 월간 / 주간 순위 표시 (Top 100)
- 1위~10위: 월말에 보너스 포인트 (1위 5,000P, 10위 500P)
- 랭크 도달 뱃지 (골드/실버/브론즈에서 진화)

**주의사항**:
- 봇 방지 필수 (다중계정으로 자기들끼리 경쟁 방지)
- Device fingerprinting 강화 + IP 기반 추가 검증

**구현 복잡도**: 중간 (리더보드 캐싱, 일일 랭킹 계산)

---

### 3. "포인트 거래소" (P2P 선물)
**목표**: 사용자 간 상호작용, 바이럴 증가

**개요**:
- 친구에게 포인트 선물 가능 (최소 100P, 월 3회)
- 선물 수신 시 "감사 포인트" 10P 추가 (인센티브)
- 선물 QR 코드로 트래킹

**위험요소**:
- 계정 간 포인트 환전 시도
- 거래 기록 위변조

**구현 복잡도**: 높음 (거래 검증, 감시)

---

### 4. "내 매장" 기능 (일반 사용자)
**목표**: UGC 확대, 크라우드소싱

**개요**:
- 일반 사용자도 매장 정보를 추가 제안 가능
- 제안이 관리자 승인되면 200P 리워드
- 커뮤니티 기반 매장 정보 축적

**위험요소**:
- 스팸/가짜 정보 증가
- 거짓 광고 처리

**구현 복잡도**: 중간 (승인 워크플로우)

---

### 5. "회원 추천" 시스템 (바이럴 루프)
**목표**: CAC 감소, 바이럴 성장

**개요**:
- 친구 초대 링크 (매장 근처 위치 기반)
- 초대자: 친구가 첫 포인트 획득 시 500P
- 초대받은 자: 초대자와의 연결로 추가 이벤트 가능

**트래킹 방식**:
- referral_code를 통한 추적
- Device fingerprinting으로 사기 방지

**구현 복잡도**: 중간

## 📈 포인트 경제 재균형 제안

### 현재 문제
- 기프티콘 3,000P가 너무 높음 (일일 10~20P로는 150일 필요)
- 출금은 7일 제한 + 1,000P 최소 (일반 사용자는 거의 접근 불가)

### 개선안 1: 중간 목표 도입
```
[낮음] 1,000P → "작은 보상" (500원 편의점 크레딧)
[중간] 5,000P → "기프티콘" (스타벅스 5,000원)
[높음] 10,000P → "출금" (현금 10,000원, 7일 대기)
[매우높음] 50,000P → "VIP 혜택" (월간 특별 전단지 접근)
```

### 개선안 2: 일일 포인트 한도 제거 + 주간 한도 추가
- 현재: 제한 없음 (봇이 하루에 1,000P 긁을 수 있음)
- 제안: 일일 200P, 주간 1,000P 한도 (너무 많이 버는 것 방지)

### 개선안 3: 포인트 만료 제도 도입
- 1년 미사용 포인트는 자동 소멸
- 이를 통해 총 포인트 순환 속도 조절

## 🎯 자영업자(Business) 경험 개선

### 현재 상태
- AdminPage에서만 전단지 관리 가능
- 포인트 예산 설정 후 직접 증액할 방법 없음
- 성과 대시보드 부족

### 개선안
1. **사업자 대시보드 별도 페이지** (MyPage와 분리)
   - 전단지 목록 (생성/수정/삭제)
   - 실시간 통계 (이 전단지 누가 스캔했나)
   - 포인트 예산 충전 (신용카드/계좌이체)
   - 수익 정산 (포인트 사용액의 10% 수수료로 현금 전환?)

2. **포인트 예산 자동 충전**
   - 예산 <50%일 때 자동 알림
   - "1,000P 충전" 원버튼 결제

3. **A/B 테스트 도구**
   - "퀴즈 포인트 10P vs 20P" 비교
   - "이번 주 긁기 수 vs 저번 주" 성장율 표시
